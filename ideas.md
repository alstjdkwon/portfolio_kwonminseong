# Chrome PDF Viewer 스타일 포트폴리오 뷰어 디자인

## 선택된 디자인 철학: **Minimal Utility Interface**

사용자가 제시한 Chrome PDF 미리보기 UI를 정확히 모방하되, 현대적이고 기능적인 인터페이스를 구현합니다.

### 핵심 원칙

1. **기능 우선**: 모든 UI 요소는 실제 기능을 수행하며, 장식적 요소는 최소화
2. **명확한 계층 구조**: 도구 모음, 사이드바, 메인 뷰어 영역이 명확히 구분
3. **다크 톤 기반**: 어두운 배경에서 콘텐츠를 돋보이게 하는 전문적 느낌
4. **응답형 레이아웃**: 모바일부터 데스크톱까지 자연스러운 적응

### 색상 철학

- **배경**: 다크 그레이(`#2d2d2d`) - 장시간 사용 시 눈 피로 감소
- **텍스트**: 밝은 회색(`#e0e0e0`) - 충분한 명도 대비
- **강조색**: 파란색(`#4a9eff`) - 상호작용 요소 강조
- **테두리**: 중간 그레이(`#404040`) - 영역 구분

### 레이아웃 구조

```
┌─────────────────────────────────────────────┐
│ 도구 모음 (상단 고정)                         │
│ ┌─────┬─────────────────────────────────────┤
│ │     │                                     │
│ │사이 │  메인 뷰어 영역                      │
│ │드바 │  (iframe으로 포트폴리오 표시)       │
│ │     │                                     │
│ │     │                                     │
│ └─────┴─────────────────────────────────────┤
└─────────────────────────────────────────────┘
```

### 시그니처 요소

1. **썸네일 네비게이션**: 왼쪽 사이드바에 페이지 썸네일 표시 (Chrome PDF와 동일)
2. **상단 도구 모음**: 페이지 이동, 줌, 회전, 다운로드 등의 버튼
3. **페이지 인디케이터**: 현재 페이지/전체 페이지 표시

### 상호작용 철학

- **호버 효과**: 버튼과 썸네일에 미묘한 배경색 변화
- **클릭 피드백**: 활성 상태 명확한 표시
- **스무스 전환**: 페이지 전환 시 부드러운 애니메이션

### 애니메이션

- **썸네일 선택**: 0.2초 fade-in으로 활성 상태 표시
- **페이지 전환**: 0.3초 opacity transition
- **줌 변화**: 0.15초로 부드러운 스케일 변화

### 타이포그래피

- **헤더/버튼**: `system-ui` 또는 `Segoe UI` (명확성)
- **페이지 번호**: 모노스페이스 폰트 (정렬감)
- **크기**: 기본 14px, 헤더 12px, 페이지 번호 13px

---

## 구현 세부사항

### 주요 기능

1. **페이지 네비게이션**: 이전/다음 버튼, 페이지 입력 필드
2. **줌 컨트롤**: 줌 인/아웃, 전체 너비/높이 맞춤
3. **회전**: 시계 방향 회전 (90도씩)
4. **사이드바 토글**: 썸네일 표시/숨김
5. **전체화면**: 몰입형 보기

### 반응형 설계

- **데스크톱**: 사이드바 240px, 도구 모음 높이 48px
- **태블릿**: 사이드바 180px, 축소된 아이콘
- **모바일**: 사이드바 숨김 (토글 버튼으로 표시), 도구 모음 축소
